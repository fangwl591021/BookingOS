# PROJECT_BRIEF.md

## 產品名稱

預約服務通（BookingOS）

## 產品定位

提供整復推拿、美髮、美甲、美睫及其他服務型店家的簡易預約系統。

## 核心原則

客人操作簡單，店家操作簡單，不追求複雜功能。第一階段以打通可交付預約流程、店家後台、必要 LINE 串接與平台管理為優先。

## 主要使用者

- 客戶：透過手機預約服務、查詢會員資料、查看點數與預約紀錄。
- 店家：設定店家資料、服務項目、服務人員、營業規則，查看預約與 CRM。
- 平台管理者：管理店家、試用、方案、收款訂單、申請審核與平台 LINE OA。

## 客戶預約主流程

```text
進入預約頁面
→ 選擇店家或分店
→ 選擇服務
→ 選擇服務人員
→ 查看可預約日期
→ 選擇時段
→ 填寫或確認資料
→ 送出預約
→ 顯示預約成功
```

目前程式實作的客戶端入口為 `/book?tenant=<tenant_id>`，由 Cloudflare Worker 直接輸出 HTML 與前端腳本。

## 特殊預約邏輯

### 指定服務人員

客戶可以指定服務人員，也可選擇「系統安排」。目前 API 使用 `staffId` 控制指定人員，`any` 代表系統安排。

### 只等指定人員

產品方向：

- 不推薦其他人。
- 顯示該人員後續可預約時段。
- 類似指定醫師掛號。
- 可往後查看該人員有空的日期。

目前狀態：尚未形成完整 UX 與 API 規則，列入下一 Sprint。

### 可接受其他安排

產品方向：

- 店家可設定是否啟用。
- 指定人員沒有時段時，詢問客戶是否願意由店家協助安排其他人員。
- 必須由客戶明確同意。
- 不可自動更換服務人員。

目前狀態：尚未完成，列入下一 Sprint。

## 店家後台基本功能

目前已有部分或完整程式：

- 店家基本資料與 LOGO。
- 服務項目、時長價格、點數折抵上限。
- 服務人員與可服務項目勾選。
- 營業時間、公休日、打烊超時規則。
- 場地資源數。
- CRM 與客戶名單下載。
- 店家端 LINE OA 設定欄位。

仍需補強：

- 人員排班。
- 預約狀態管理與取消流程的店家後台操作。
- 指定人員與替代安排規則。
- 權限與登入安全。

## Admin 總管理後台

目前已有 `/platform`，包含：

- 店家管理。
- 試用列表。
- 收款訂單。
- 申請審核。
- 費用方案連結。
- 平台 LINE OA 參數。
- 好友 CRM。
- Webhook 紀錄。

## 第一階段暫不處理

除非現有程式已經完成，否則暫不新增：

- 金流自動串接。
- 優惠券。
- AI 排班。
- 多種社群登入。
- 原生 App。
- WordPress 深度整合。
- 過度複雜的報表。

注意：目前程式已存在點數、介紹人、CRM、收款訂單雛形；本文件不要求本輪擴充。

## 成功標準

- 客戶可完成一筆預約，資料寫入 D1。
- 店家可查看該店預約與基本資料。
- 多店資料以 `tenant_id` 隔離，不互相混用。
- 平台後台可管理店家、試用、方案與 LINE OA 設定。
- LINE Login、Messaging API、Webhook 的用途與設定位置清楚。
- 可部署至 Cloudflare Workers，並有 `.env.example` 與專案狀態文件。
